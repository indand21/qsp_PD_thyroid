# QSP PD Thyroid Model

Quantitative systems pharmacology model for checkpoint inhibitor-associated thyroid dysfunction. The repository contains the model implementation, configuration system, analysis pipeline, validation tests, and mathematical-methods documentation.

## Project Layout

- `qsp_expanded/` - the current expanded QSP model package (ODE system, parameters, PK, simulation, and thyroid-status classification); this is the actively-used model.
- `scripts/run_v1_pipeline.py` - orchestrator for the v1 analysis pipeline, with shared helpers `scripts/v1_common.py` and `scripts/cohort_features.py`.
- `scripts/{calibration,analysis,sensitivity,cohorts,population,validation,ml,figures}/` - the individual v1 pipeline stage scripts run by the orchestrator.
- `config/`, `utils/` - configuration/parameter and utility modules.
- `tests/` - unit, performance, and fixture tests (see `tests/unit/test_expanded_model.py` for the current model).
- `docs/` - mathematical-methods documentation: the governing model equations and the supplementary mathematical methods.
- `results/v1/` - outputs generated by the v1 pipeline (calibration, incidence, validation, virtual population, ML, figures). Regenerable; the generated artifacts are gitignored.

## Common Commands

```powershell
python -m pytest tests/unit -q
python scripts/run_v1_pipeline.py
```

The `Makefile` documents additional workflows for testing, linting, Docker builds, and validation. Some targets use Unix shell commands and may need PowerShell equivalents on Windows.

## Maintenance Notes

Generated caches, logs, benchmark outputs, and top-level analysis artifacts are ignored by `.gitignore`. Pipeline outputs live under `results/v1/` and are regenerated by `python scripts/run_v1_pipeline.py`.
